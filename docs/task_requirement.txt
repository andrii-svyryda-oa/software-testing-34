Завдання 34: Програма лояльності
Предметна область
Система лояльності клієнтів з балами, нагородами та їх використанням.

Сутності
Customer: Id, Name, Email, Phone, TierLevel (Bronze/Silver/Gold/Platinum), TotalPoints, JoinDate
PointTransaction: Id, CustomerId, Points, Type (Earned/Redeemed/Expired/Bonus), Description, CreatedAt
Reward: Id, Name, Description, PointsCost, Category, StockQuantity, IsActive
Ендпоінти
Метод	Маршрут	Опис
GET	/api/customers/{id}	Отримати клієнта з балансом балів
POST	/api/customers	Зареєструвати клієнта
POST	/api/customers/{id}/earn	Додати зароблені бали
POST	/api/customers/{id}/redeem	Обміняти бали на нагороду
GET	/api/customers/{id}/history	Отримати історію транзакцій балів
GET	/api/rewards	Отримати доступні нагороди
POST	/api/rewards	Створити нагороду
GET	/api/customers/{id}/tier	Отримати статус рівня та прогрес
Бізнес-правила
Рівень на основі загальної кількості зароблених балів: Bronze(0), Silver(1000), Gold(5000), Platinum(10000)
Не можна обміняти більше балів, ніж є на балансі
Нагорода повинна бути в наявності для обміну
Бали згорають через 12 місяців, якщо не використані
Учасники Gold+ отримують 1.5x балів; Platinum отримує 2x
Обсяг тестування
Модульні тести: Розрахунок рівня, логіка множника балів, правила закінчення терміну дії
Інтеграційні тести (WebApplicationFactory): Процес нарахування/обміну, підвищення рівня, використання нагороди
Тести бази даних (Testcontainers): Консистентність балансу балів, історія транзакцій, зменшення запасу нагород
Тести продуктивності (k6): Навантажувальне тестування запитів балансу балів, стрес-тестування одночасних обмінів
Поля AutoFixture
Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

Customer: Name, Email, Phone, JoinDate
PointTransaction: Description, CreatedAt
Reward: Name, Description, Category
Поля TierLevel, TotalPoints, Points, Type, PointsCost, StockQuantity, IsActive, CustomerId мають бізнес-правила і повинні встановлюватися явно в тестах.

База даних
Використовуйте PostgreSQL як двигун бази даних. Для Testcontainers використовуйте пакет Testcontainers.PostgreSql.

Наповнення бази даних
Для тестів продуктивності та інтеграційних тестів база даних повинна бути попередньо наповнена щонайменше 10 000 записами, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Приклад розподілу повинен відображати реальні співвідношення між батьківськими та дочірніми сутностями.
